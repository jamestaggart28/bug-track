Introduction

Bug track is an error tracking system made for developers to record all known bugs in their code. Inspired by The Joel Test, bug track was meant to add a more specialized approached to the traditional ‘bug database’. What makes bug track special for developers is that it takes into consideration modularity. To understand that you must understand the difference between projects and apps.

Projects

Projects are web-based systems that consist of multiple applications. Projects refer to the entire application and all of its parts.

Apps

Apps refer to all the reusable components that make up a project. Apps should be self-contained components you could pick up and place into another project without much, or any, modification.

What is the Point

The purpose and what distinguishes bug track from your traditional bug database is that it tracks the relation between projects and their applications so if later a reusable application gets a bug you can track all the affected projects. The benefits of tracking this simple relation can be tremendous. If you have an application that spans across many projects and that application has a bug you are no longer left in the dark on which projects were affected. Giving you full accountability.
